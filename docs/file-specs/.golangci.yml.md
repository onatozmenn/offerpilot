# File Spec: `.golangci.yml`

## Status

`validated`

## Purpose

Configure reproducible Go static analysis without stylistic noise.

## Depends On

- Go 1.26 module.
- `docs/09-testing.md`.

## Public Surface

- Repository lint policy consumed by `golangci-lint run`.

## Required Behavior

- Enable correctness-focused checks for errors, body closure, context misuse, SQL/resource handling, unnecessary conversions, and standard vet/staticcheck suites.
- Exclude generated files and allow narrowly documented test exceptions.
- Set a practical timeout and use the configuration schema supported by the pinned CI linter.

## Failure Cases

- Deprecated linter names, blanket exclusions, or a configuration that skips tests.

## Non-Goals

- Replacing `gofmt`, tests, or the race detector.

## Validation

- `golangci-lint config verify`
- `golangci-lint run`

## Completion Checklist

- [ ] Configuration is accepted by the pinned linter.
- [ ] No broad suppressions exist.
- [ ] Focused validation passes.
