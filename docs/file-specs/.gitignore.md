# File Spec: `.gitignore`

## Status

`specified`

## Purpose

Exclude local secrets, dependency outputs, binaries, coverage, editor state, and generated frontend artifacts.

## Depends On

- `docs/generated-files.md`
- Planned Go, npm, Docker, and environment workflows.

## Public Surface

- Repository-wide ignore rules.

## Required Behavior

- Ignore `.env`, Go binaries/profiles/coverage, `web/node_modules`, `web/dist`, frontend coverage, local database volumes, OS files, and editor-local settings not intended for sharing.
- Keep `.env.example`, source migrations, lock files, and documentation tracked.
- Do not use a broad rule that hides implementation fixtures or OpenAPI artifacts.

## Failure Cases

- A local secret or generated directory appears in `git status`.
- A required lockfile or source fixture is accidentally ignored.

## Non-Goals

- Secret scanning or cleanup of already tracked files.

## Validation

- Create representative temporary paths, run `git check-ignore -v`, then remove the temporary files.

## Completion Checklist

- [ ] Required local/generated paths are ignored.
- [ ] Required source artifacts remain trackable.
- [ ] Focused validation passes.
